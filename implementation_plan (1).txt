PART 1: MAIN WEBSITE UPGRADE PLAN
--------------------------------------------------------------------------------

GOAL:
Transform the KMIT website into an award-worthy digital experience with 
production-grade performance, accessibility, and modern visual design.

CURRENT STATE AUDIT & GAPS:
1. Hero Section & Slider:
   - Static text, missing conversion CTAs.
   - Low contrast overlay competing with text.
   - Missing slide indicators & 5s timer progress bar.
2. Section Design & Interactivity:
   - StatsBar lacks glassmorphism and animated glow effects.
   - PlacementSection data hardcoded in JSX instead of CMS hook.
   - WhyChooseSection lacks 3D depth/perspective hover effects.
   - EventsSection needs lazy-load shimmer and dynamic captions.
   - ChatbotWidget contains injected style tags & lacks keyword matching.
3. Code Quality & Performance:
   - index.css size (~191 KB) with unoptimized dead code.
   - Lacks SEO meta tags, OpenGraph, Twitter Cards, and preconnect links.
   - Lacks route-level ErrorBoundary & 404 page fallback.

PHASED IMPLEMENTATION:

Phase 1 — Foundation Fixes (Performance & Code Health)
------------------------------------------------------
- index.html: Add preconnect for Google Fonts/FontAwesome, SEO meta tags, OpenGraph, Twitter cards.
- vite.config.js: Add build optimizations (es2020 target, cssCodeSplit, assetsInlineLimit, compression).
- index.css: Clean up dead CSS, refine color palette (--lime fix), add prefers-color-scheme dark mode support.

PART 2: ADMIN PANEL IMPLEMENTATION PLAN
--------------------------------------------------------------------------------

GOAL:
Implement a robust, secure, role-based Admin Portal for dynamic management 
of website content, notices, placements, faculty, and academic files.

1. System Architecture:
   - Database: MongoDB with Mongoose schemas (User, Department, Faculty, Notice, Page, AcademicDoc, SiteData, AuditLog).
   - Storage: Media stored in server/uploads or S3, DB stores URL references.
   - Authentication: JWT tokens with HTTP-only cookies / local storage.
   - RBAC Roles: Super Admin, Department HOD, Content Editor.

2. Content Management Scope:
   - Academics & Faculty: Faculty profiles, department details, syllabi, exam schedules.
   - Announcements: Homepage ticker, circulars, urgent alerts.
   - Placements: Recruiter logos, placement statistics, student testimonials.
   - Static/Dynamic Pages: Department pages, committee pages, general info.
   - Site Configuration: Contact details, footer links, social links.

3. Safeguards & Immutability:
   - Restricted CSS/Layout changes: Design tokens, layout structure, and fonts remain locked.
   - Audit Log: Every action logged with append-only security (no deletion permitted).
